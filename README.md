\# Student Affairs Simulation System 🎓



A Java-based simulation engine designed to manage and process student queues in a Student Affairs office. This project demonstrates the practical application of core Data Structures (Linked Lists \& Queues) and Object-Oriented Programming (OOP).



\## 🚀 Features

\- \*\*Dynamic Queue Management:\*\* Efficiently adds students to the shortest available line.

\- \*\*Real-time Simulation:\*\* Processes students based on assigned processing time per minute.

\- \*\*Search \& Delete:\*\* Allows searching for students by ID or removing them from any point in the queue.

\- \*\*Performance Tracking:\*\* Monitors algorithm efficiency through "Comparisons" and "Swaps" counters.



\## 🛠️ Data Structures Used

\- \*\*Linked List:\*\* Custom implementation for dynamic memory management.

\- \*\*Queue:\*\* Built on top of the Linked List to follow FIFO (First-In-First-Out) logic.



\## 📊 Statistics

The system tracks:

\- \*\*Comparisons:\*\* Number of search operations performed.

\- \*\*Swaps/Modifications:\*\* Total changes made to the data structure pointers.



\## 📝 How to Run

2\. Compile and run `SemesterProject.java`.

3\. Follow the on-screen menu instructions.
